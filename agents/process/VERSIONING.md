# Versioning
Date: 2026-02-03

We use Semantic Versioning 2.0.0 (SemVer).

## Format
MAJOR.MINOR.PATCH

## Rules
- MAJOR: incompatible API changes
- MINOR: backward-compatible functionality
- PATCH: backward-compatible bug fixes

## Guidance
- Pre-release tags (e.g., 1.2.0-rc.1) are allowed when needed.
- If unsure, default to PATCH and document the rationale.
